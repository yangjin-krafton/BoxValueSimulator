# Box Value Simulator

택배 상자를 개봉해 랜덤한 상품을 획득하고,  
상품의 상태(퀄리티/완성도)에 따라 등급과 판매 가격이 결정되는  
웹 기반 캐주얼 루프형 언박싱 시뮬레이션 게임입니다.

이 프로젝트는 **PixiJS 기반 렌더링**을 사용하고,  
**3D 모델 에셋을 2.5D/의사 3D 방식으로 활용**하는 것을 목표로 합니다.

---

## 게임 컨셉

플레이어는 다양한 크기와 가격대의 택배 상자 중 원하는 상자를 선택해 개봉합니다.  
상자 안에는 랜덤한 상품이 들어 있으며, 상품은 3D 모델로 표현됩니다.

획득한 상품은 셰이더 기반 텍스처 상태 변화, 표면 오염, 마모, 흠집, 색편차 등의 연출을 통해  
각기 다른 상태값을 가지게 되고, 이 상태에 따라 등급이 부여됩니다.

등급은 아래와 같은 단계로 구성됩니다.

- C
- B
- A
- S
- SS
- SSS

최종 판매 가격은 다음 요소를 조합해 계산됩니다.

- 상품의 원래 기준 가치
- 현실 세계 물가를 참고한 가격 환산값
- 상품 상태 등급 보정
- 희소도 또는 특수 보너스

플레이어는 상품을 판매해 자금을 얻고,  
그 돈으로 다시 새로운 상자를 구매해 개봉하는  
**무한 루프형 가챠/언박싱 경제 구조**를 반복하게 됩니다.

---

## 핵심 게임 루프

1. 상자 목록이 제공된다.
2. 플레이어가 구매할 상자를 선택한다.
3. 상자를 개봉한다.
4. 랜덤한 3D 상품을 획득한다.
5. 상품 상태가 셰이더/텍스처 배리에이션으로 결정된다.
6. 등급과 판매 가격이 산출된다.
7. 상품을 판매해 보상을 얻는다.
8. 다시 더 비싼 상자 또는 새로운 상자에 도전한다.
9. 자금이 바닥나면 반복 노동 미니게임으로 기본 자금을 복구한다.
10. 다시 상자 개봉 루프로 진입한다.

## 상자 시스템

게임에서는 한 번에 **10개의 상자 세트**가 제공됩니다.

이 10개 상자는 단순히 각각 독립적인 랜덤 가치가 있는 것이 아니라,  
먼저 **전체 10개 상자 내부 상품의 총 가치 합산**이 결정되고,  
그 뒤 이 총 가치가 각 상자에 랜덤하게 분배됩니다.

즉,

- 어떤 상자는 매우 비싸지만 꽝에 가까울 수 있고
- 어떤 상자는 싸지만 내부 가치가 매우 높을 수 있으며
- 유저는 제한된 정보 안에서 상자를 선택해야 합니다

이 방식은 단순 확률형 가챠가 아니라,  
**같은 세트 안에서 총합은 정해져 있지만 분포가 랜덤한 위험/보상 구조**를 만듭니다.

### 설계 의도

- 플레이어에게 “이번 세트는 총량이 좋다 / 나쁘다”는 감각 제공
- 싼 상자에서 대박이 나는 경험 제공
- 비싼 상자에서 손해를 보는 리스크도 허용
- 단순 확률 뽑기가 아니라 선택과 기대 심리를 강화

---

## 상품 시스템

상자 안에서 등장하는 상품은 다양한 카테고리로 구성될 수 있습니다.

예시:

- 전자기기
- 피규어
- 생활용품
- 장난감
- 레트로 소품
- 명품 느낌의 잡화
- 수집품
- 정체불명의 희귀 아이템

각 상품은 아래 데이터를 가질 수 있습니다.

- 상품 이름
- 기본 가치
- 희소도
- 카테고리
- 3D 모델 리소스
- 상태 파라미터
- 등급
- 최종 판매가

---

## 상태 등급 시스템

같은 상품이라도 상태에 따라 가격이 크게 달라집니다.

예를 들어 같은 모델이라도 다음과 같은 차이가 발생할 수 있습니다.

- 표면 스크래치
- 먼지/오염 정도
- 포장 손상
- 색상 편차
- 광택 손실
- 일부 흠집 또는 파손 느낌
- 반대로 매우 완벽한 보존 상태

이 상태값은 셰이더와 텍스처 배리에이션으로 시각화합니다.

### 등급 예시

| 등급 | 의미 | 가격 배수 예시 |
|------|------|----------------|
| C    | 상태 불량 | x0.4 |
| B    | 사용감 많음 | x0.7 |
| A    | 평균 이상 | x1.0 |
| S    | 매우 양호 | x1.4 |
| SS   | 희귀급 상태 | x2.0 |
| SSS  | 최고급 보존 상태 | x3.5 |

등급은 단순 랜덤이 아니라,  
상품별 상태 파라미터와 확률 테이블에 의해 결정됩니다.

---

## 판매 가격 계산 방식

최종 판매 가격은 대략 다음 공식을 따릅니다.

```text
판매가 = 기본가치 × 상태등급배수 × 희소도배수 × 시장보정값
````

예시:

```text
기본가치: 32,000원
상태등급: SS = x2.0
희소도배수: x1.3
시장보정값: x0.95

최종 판매가 = 32,000 × 2.0 × 1.3 × 0.95
            = 79,040원
```

### 가격 설계 목표

* 현실 물가 감각을 반영해 직관적인 가치 판단 가능
* 높은 등급의 쾌감을 크게 제공
* 단, 무한 인플레이션이 발생하지 않도록 상자 가격과 판매가를 통제
* 초반에는 소액 보상 중심, 후반에는 고가 리스크 중심 구조 형성

## 경제 구조

이 게임의 핵심은 **상자 개봉 → 판매 → 재투자**의 경제 루프입니다.

하지만 운이 나쁘면 플레이어의 자금이 모두 소진될 수 있으므로,
완전한 게임 오버 대신 다시 진입 가능한 **기본금 회복 루프**가 필요합니다.

### 파산 방지 장치

플레이어의 소지금이 바닥나면
단순 반복 노동 미니게임을 통해 기본 자금을 얻을 수 있습니다.

예시:

* 상품 포장하기
* 송장 붙이기
* 박스 분류하기
* 택배 상차 보조
* 라벨 정렬하기

이 미니게임은 높은 수익을 주지 않지만,
다시 상자를 구매할 최소 자금을 안정적으로 회복하게 만듭니다.

### 경제 설계 목표

* 파산은 있어도 영구 탈락은 없음
* 운이 나쁘면 노동 루프로 복귀
* 운이 좋으면 고가 상자까지 도전 가능
* 리스크와 회복 루프가 함께 존재
* 반복 플레이 동기 유지

## 내부 수학 모델 설계 방향

이 프로젝트는 단순 랜덤이 아니라,
플레이 가능한 장기 루프를 만들기 위한 수학 모델이 중요합니다.

### 1. 10상자 세트 총량 모델

한 세트에 포함된 10개 상자의 총 기대 가치를 먼저 생성합니다.

예시:

* 세트 총 가치 = 180,000원 ~ 320,000원 범위
* 이 값을 10개 상자에 비대칭 분배
* 분배 시 일부 상자는 기대값 이하, 일부 상자는 기대값 이상

이를 통해 플레이어는 다음과 같은 경험을 하게 됩니다.

* “이번 줄은 별로인 것 같다”
* “이 박스는 싸지만 뭔가 있을 것 같다”
* “비싼 박스가 꼭 좋은 건 아니다”

### 2. 상자 가격 분포 모델

상자 가격은 내부 실제 가치와 완전히 같지 않게 설계합니다.

즉,

* 실제 기대값보다 비싸게 책정된 함정 상자
* 실제 기대값보다 싸게 책정된 기회 상자
* 평균적으로는 약간 손해 구조
* 드물게 큰 이득 발생

이렇게 해야 가챠의 기대와 긴장감이 살아납니다.

### 3. 상품 상태 모델

상품 획득 후 상태가 결정될 때
완전 균등 확률이 아니라 상품군에 맞는 분포를 사용합니다.

예시:

* 저가 상품: B/A 비중 높음
* 프리미엄 상품: C~SSS까지 폭 넓음
* 희귀 상품: 등장 확률은 낮지만 상위 등급 보정 큼

### 4. 장기 기대값 통제

플레이어가 계속 플레이할 수 있도록
전체 기대 수익률은 아래 중간 어디쯤에 맞추는 것이 좋습니다.

* 평균 1회 상자 구매 시 기대 회수율: **0.82 ~ 0.95**
* 좋은 선택/운 좋은 결과일 때 고수익 가능
* 장기적으로는 소폭 손해 구조
* 파산 시 노동 루프로 회복 가능

이 구조는 다음을 동시에 만족시킵니다.

* 게임이 쉽게 붕괴하지 않음
* 초반 자금이 너무 빨리 사라지지 않음
* 대박 경험이 존재함
* 노동 루프가 게임 전체를 지탱함

## 아트 및 비주얼 방향

이 프로젝트는 “진짜 3D 실시간 게임”보다
**3D 모델을 활용한 2.5D 스타일**에 가깝습니다.

### 비주얼 구성

* 상자: 여러 크기와 테이프/라벨 디자인
* 상품: 3D 모델 기반 스프라이트 또는 렌더 이미지
* 상태 표현: 셰이더, 색 변화, 오염, 흠집, 광택 차이
* 개봉 연출: 박스 흔들림, 절개, 펼침, 등장 이펙트
* 판매 연출: 가격 카운팅, 등급 연출, 보상 팝업

### 권장 방식

PixiJS는 고성능 2D 렌더러이므로,
3D 모델은 다음 방식으로 사용하는 것이 현실적입니다.

* Blender 등에서 3D 모델 렌더링
* 스프라이트 시트 또는 방향별 이미지 추출
* PixiJS에서 표시
* 셰이더 필터로 상태 변화 추가

---

## 기술 스택

* **Renderer**: Three.js (CDN via importmap, 실시간 3D, GLB via GLTFLoader)
* **Language**: Vanilla JavaScript (ES modules, No TypeScript, No bundler)
* **Deployment**: GitHub Pages — `src/` 디렉토리 직접 서빙, 빌드 스텝 없음
* **State**: EventBus + GameStateManager (커스텀)
* **3D Models**: GLB 포맷, 런타임 로드; 폴백 프로시저 도형
* **Shader/Material**: PBR 10종 (MeshPhysicalMaterial) + Matcap 10종 = 20종 프리셋
* **Data**: CSV 기반 상품 테이블 (자동 생성) + JSON manifest
* **배포/서빙**: 정적 페이지 (GitHub Pages) — 서버 없이 `src/` 단독 배포

## GLB 모델 추가 및 상품 테이블 생성 가이드

새로운 3D 모델(GLB)을 추가하면, 자동 스캔 에이전트가 스크린샷 촬영 → AI 분석 → 20종 셰이더 변형 → CSV 테이블 생성까지 처리합니다.

### 사전 요구사항

* **Node.js** (v18 이상)
* **LM Studio** — Vision 지원 모델 로드 (예: `qwen/qwen3.5-9b`)
* LM Studio 서버 실행 상태 (`http://<IP>:1234`)

### 1단계: GLB 파일 배치

```
src/assets/models/ 아래에 .glb 파일을 복사합니다.
```

파일명 규칙: `카테고리명변형번호.glb` (예: `AccessoryGlassBoston0.glb`)
같은 디자인의 색상 변형은 번호만 변경 (0, 1, 2...)

### 2단계: 스캔 에이전트 실행

```bash
cd tools
npm install          # 최초 1회
npm run scan         # 전체 스캔 (이미 처리된 모델은 자동 건너뜀)
```

### 주요 옵션

| 플래그 | 기본값 | 설명 |
|--------|--------|------|
| `--lm-url` | `http://100.66.10.225:1234` | LM Studio 서버 주소 |
| `--model` | `qwen/qwen3.5-9b` | Vision 모델 이름 |
| `--batch` | `5` | 배치 크기 (N개씩 LM에 한 번에 전송) |
| `--reset` | - | 기존 CSV 무시하고 처음부터 다시 |
| `--dry-run` | - | CSV 파일 안 쓰고 결과만 출력 |

### 에이전트 파이프라인 (5단계)

```
Stage 1: Scanner     — src/assets/models/*.glb 스캔
Stage 2: Renderer    — Puppeteer + Three.js로 512×512 스크린샷 캡처
Stage 3: Analyzer    — LM Studio Vision에 배치 전송, 한국어 상품명/가격/희귀도 산출
Stage 4: Preset Gen  — 모델당 20종 셰이더 변형 (PBR 10 + Matcap 10) 생성
Stage 5: CSV 저장    — 5개 모델 처리할 때마다 CSV에 점진적 추가 (중복 체크)
Stage 6: CSV 분할    — 200행 단위로 분할 + products-manifest.json 생성
```

### 3단계: 결과 확인

```
src/data/
├── products-manifest.json      # CSV 파일 목록 (게임 로더가 참조)
├── products.csv                # 전체 마스터 CSV
├── products_01.csv ~ _NN.csv   # 200행 단위 분할 파일
├── presets.js                  # PBR 10 + Matcap 10 프리셋 정의
```

CSV 컬럼: `id, name, baseValue, rarity, category, modelPath, preset`

### 20종 셰이더 프리셋

각 모델은 아래 20종 재질 변형으로 확장됩니다.
baseValue에 valueMod%가 적용되어 변형별 가격이 달라집니다.

**PBR 10종** (원본 텍스처 유지 + MeshPhysicalMaterial 파라미터 override):

| 프리셋 | 라벨 | 가치 보정 |
|--------|------|-----------|
| plastic_matte | 매트 플라스틱 | +0% |
| plastic_gloss | 글로시 플라스틱 | +10% |
| rubber_soft | 소프트 러버 | +5% |
| ceramic_clean | 클린 세라믹 | +20% |
| metal_brushed | 브러시드 메탈 | +35% |
| metal_polished | 폴리시드 메탈 | +50% |
| paint_clearcoat | 클리어코트 페인트 | +30% |
| fabric_sheen | 패브릭 쉰 | +15% |
| glass_clear | 클리어 글래스 | +60% |
| resin_tinted | 틴티드 레진 | +45% |

**Matcap 10종** (MeshMatcapMaterial로 교체, 조명 불필요):

| 프리셋 | 라벨 | 가치 보정 |
|--------|------|-----------|
| matcap_clay | 클레이 | +5% |
| matcap_wax | 왁스 | +10% |
