# Box Value Simulator 개선 기획

## 1. 문서 목적

이 문서는 현재 게임의 실제 구조를 기준으로, 상자 가격 결정과 내용물 결정 로직을 재설계하고 이를 `라운드 진열 시스템`과 연결하기 위한 기준 문서다.

현재 게임의 핵심 전제는 다음과 같다.

- 한 라운드는 상자 10개 세트다.
- 상자는 2~4개 열로 랜덤 적층된다.
- 유저는 각 열의 최상단 상자만 구매할 수 있다.
- 유저는 개봉한 상품을 `즉시 판매`하거나 `진열 슬롯`에 올릴 수 있다.
- 진열 슬롯은 3칸이며, 슬롯 판매가 총합으로 라운드 목표를 달성해야 한다.

즉 이 게임은 더 이상 단순한 개봉 후 즉시 판매 게임이 아니라, `열 선택 + 개봉 + 3슬롯 핸드 운영` 게임이다.

## 2. 현재 구현 구조 요약

### 2.1 세트 생성

현재 `src/systems/BoxGenerator.js`는 아래 순서로 동작한다.

1. 세트 전체 가치 총합을 정한다.
2. 그 총가치를 10개 상자에 랜덤 비율로 나눠 `innerValue`를 만든다.
3. 각 `innerValue`와 가까운 상품을 고른다.
4. `innerValue`에 랜덤 노이즈를 섞어 상자 가격을 정한다.

즉 현재 구조는 아래와 같다.

`세트 총가치 -> 박스별 내부가치 -> 상품 선택 -> 상자 가격 생성`

### 2.2 진형과 구매 규칙

현재 `src/scenes/BoxSelectionScene.js`에서 다음 규칙이 적용된다.

- 10개 상자를 2~4개 tower에 랜덤 분배한다.
- 각 tower의 최상단 상자만 현재 선택 가능하다.
- 상자 하나를 열면 그 아래 상자가 다음 선택지로 열린다.
- 특정 tower를 모두 비우면 보너스 할인 구조가 발동한다.

즉 유저는 항상 "지금 열린 최상단 후보들" 중에서 다음 상자를 결정한다.

### 2.3 개봉 시점 결정

현재 `src/scenes/UnboxingScene.js`는 개봉 직전에 다음을 수행한다.

- 이미 박스에 연결된 상품을 가져온다.
- 등급은 `rollGrade()`로 새로 뽑는다.
- 판매가는 `상품 기본가 x 등급 배율 x rarity x 시장 보정`으로 계산한다.

즉 현재는 상품은 세트 생성 시점에 거의 정해지고, 개봉 시점에는 등급만 추가되는 구조다.

## 3. 현재 구조의 문제

### 3.1 상자 가격 규칙이 유저에게 읽히지 않는다

상자 가격은 내부적으로 `innerValue` 기반이지만, 유저가 보기에는 랜덤 가격처럼 보인다.
즉 규칙이 플레이 판단 정보로 공개되지 않는다.

### 3.2 세트 구조가 경제 설계에 충분히 활용되지 않는다

게임의 재미는 `어느 열을 먼저 밀지`인데, 현재 경제 설계는 열의 역할과 깊이를 적극 활용하지 않는다.

### 3.3 비싼 상자의 의미가 약하다

비싼 상자는 아래를 기대하게 만들어야 한다.

- 더 높은 기본 상품가 범위
- 더 높은 상위 등급 확률
- 더 높은 히든 확률
- 더 높은 보너스 기대값

하지만 현재는 이 연결이 명확하지 않다.

### 3.4 개봉 후 판단 구조가 약하다

이제 게임 목표는 `좋은 상품을 많이 파는 것`이 아니라, `좋은 상품을 골라 3슬롯으로 라운드를 넘기는 것`까지 포함한다.
기존 구조는 이를 전제로 짜여 있지 않다.

## 4. 재설계 방향

핵심 방향은 단순하다.

- 상자 10개 세트 구조는 유지한다.
- 2~4열 랜덤 적층 구조도 유지한다.
- 최상단만 구매 가능한 규칙도 유지한다.
- 대신 세트 내부 경제를 `열 역할`과 `깊이 보정` 중심으로 다시 짠다.
- 개봉 후에는 `즉시 판매`와 `진열` 중 하나를 선택하게 만든다.
- 진열 슬롯 3개 총합으로 라운드 목표를 넘기도록 바꾼다.

즉 인터랙션 뼈대는 유지하고, 경제 모델과 라운드 목표를 교체한다.

## 5. 새 설계의 핵심 원칙

### 5.1 세트는 먼저 성격을 가져야 한다

세트는 단순한 상자 묶음이 아니라 하나의 판이어야 한다.
매 세트는 먼저 경제 성향을 정한다.

예시:

- 안정형 세트
- 균형형 세트
- 공격형 세트
- 잭팟형 세트
- 보너스형 세트

### 5.2 열은 역할을 가져야 한다

각 열은 단순 적층 공간이 아니라 전략 단위가 되어야 한다.

예시:

- 안정 열: 가격이 낮고 평균 회수율이 높음
- 공격 열: 가격이 높고 상위 등급 확률이 높음
- 심층 열: 위는 평범하지만 아래로 갈수록 급격히 좋아짐
- 보너스 열: 할인권, 히든, 열 클리어 보상이 강함

### 5.3 아래층일수록 기대값이 커야 한다

같은 열에서 아래층으로 갈수록 다음 요소가 상승해야 한다.

- 상품 기본가 범위
- 상위 등급 확률
- 히든 확률
- 보너스 기대값

### 5.4 공개 정보만으로 다음 선택을 판단할 수 있어야 한다

유저는 개봉 전 최상단 상자만 보기 때문에, 그 제한된 정보만으로 열 선택을 해야 한다.

최소 노출 정보:

- 구매가
- 상품가 구간
- 열 타입
- 예상 회수율
- 대박 확률

### 5.5 개봉 후에는 핸드 운영 판단이 생겨야 한다

개봉 이후에는 `팔 것인가`, `슬롯에 둘 것인가`, `기존 슬롯을 갈아낄 것인가`가 핵심이 된다.
즉 상자 선택 전략과 상품 핸드 운영 전략이 분리되어야 한다.

## 6. 현재 구조에 맞는 생성 순서

현재는 `세트 총가치 -> 상자 내부가치 -> 상품 선택`인데, 새 구조는 아래 순서가 더 적합하다.

1. 세트 타입 결정
2. tower 수 결정
3. tower별 역할 결정
4. 각 tower의 층 수 결정
5. 각 슬롯의 가격 밴드 결정
6. 각 슬롯의 기본 상품가 범위 결정
7. 그 범위에 맞는 상품 선택
8. 각 슬롯의 등급 확률표, 히든 확률, 쿠폰 확률 부여

즉 박스는 단순 가격/상품 객체가 아니라, `개봉 룰과 기대값 정보까지 가진 슬롯 객체`가 되어야 한다.

## 7. 제안하는 세트 타입

### 7.1 안정형

- 상자 가격이 전반적으로 낮음
- 평균 회수율이 높음
- 대박 확률은 낮음
- 초반 진행용

### 7.2 균형형

- 가격과 기대값이 고르게 분포
- 특정 열 편향이 적음
- 기본형 세트

### 7.3 공격형

- 일부 열 가격이 높음
- 상위 등급 확률이 높음
- 평균 회수율은 변동이 큼

### 7.4 잭팟형

- 평균 회수율은 낮지만 SSSS 이상 확률이 높음
- 큰 대박을 노리는 판

### 7.5 보너스형

- 열 클리어 보너스가 강함
- 할인권, 다음 상자 할인, 특수 효과 기대값이 높음

## 8. 제안하는 열 역할

### 8.1 안정 열

- 가격이 낮거나 중간
- 기본 상품가 구간이 가격 대비 안정적
- 플러스 수익 확률이 높음

### 8.2 공격 열

- 가격이 높음
- 기본 상품가 범위 상단이 큼
- SS 이상 등장 확률이 높음

### 8.3 심층 열

- 최상단은 평범
- 아래층으로 갈수록 눈에 띄게 좋아짐

### 8.4 보너스 열

- 할인권, 히든 등장, 열 클리어 보상에 특화

## 9. 상자 가격 결정 방식 재설계

현재처럼 `innerValue x 노이즈`로 가격을 만들면 규칙이 잘 보이지 않는다.
새 구조에서는 먼저 가격 밴드를 정하고 그 안에서 가격을 선택해야 한다.

### 9.1 가격 밴드 예시

| 밴드 | 가격 |
| --- | ---: |
| B1 | 10,000 |
| B2 | 20,000 |
| B3 | 30,000 |
| B4 | 50,000 |
| B5 | 80,000 |
| B6 | 120,000 |
| B7 | 200,000 |

### 9.2 가격 결정 원칙

- 세트 타입이 가격 분포를 조정한다.
- 열 역할이 어떤 밴드에 걸릴지 결정한다.
- 같은 열에서는 아래층으로 갈수록 상위 밴드 비중이 높아진다.

예시:

- 안정 열 상단: B1~B2
- 안정 열 하단: B2~B3
- 공격 열 상단: B3~B4
- 공격 열 하단: B4~B6

## 10. 내용물 결정 방식 재설계

내용물은 두 단계로 결정한다.

1. 기본 상품가 범위 결정
2. 개봉 시 셰이더 등급과 특수 보정 적용

### 10.1 기본 상품가 구간

상자마다 아래 데이터를 가진다.

- `baseValueMin`
- `baseValueMax`

상품은 이 범위 안에서 뽑거나, 이 범위와 가까운 후보군에서 선택한다.

예시:

| 상자 가격 | 기본 상품가 범위 |
| --- | --- |
| 10,000 | 1,000 ~ 20,000 |
| 20,000 | 5,000 ~ 35,000 |
| 30,000 | 8,000 ~ 50,000 |
| 50,000 | 10,000 ~ 100,000 |
| 80,000 | 20,000 ~ 180,000 |
| 120,000 | 35,000 ~ 300,000 |
| 200,000 | 60,000 ~ 600,000 |

### 10.2 개봉 시 적용되는 등급 배율

상품 기본가는 고정하고, 개봉 시 셰이더 등급 배율을 붙인다.

권장 배율:

| 등급 | 배율 |
| --- | ---: |
| A | 1.00 |
| S | 1.55 |
| SS | 2.40 |
| SSS | 4.20 |
| SSSS | 8.20 |
| SSSSS | 16.50 |

## 11. 등급 확률 재설계

현재는 전역 공통 확률표지만, 앞으로는 슬롯별 확률표가 필요하다.

### 11.1 기본 방향

- 싼 상자는 A~SS 중심
- 비싼 상자는 S~SSSSS 상방 확대
- 공격 열은 같은 가격대라도 상위 등급 확률이 더 높음
- 심층 열은 아래층으로 갈수록 상위 등급 확률이 소폭 증가

### 11.2 예시 확률표

#### 안정 슬롯

- A 70%
- S 20%
- SS 8%
- SSS 1.7%
- SSSS 0.25%
- SSSSS 0.05%

#### 공격 슬롯

- A 48%
- S 27%
- SS 15%
- SSS 7%
- SSSS 2.3%
- SSSSS 0.7%

#### 최심부 공격 슬롯

- A 36%
- S 28%
- SS 18%
- SSS 10%
- SSSS 5%
- SSSSS 3%

## 12. 히든 상품과 할인권

### 12.1 히든 상품

히든은 별도 풀로 관리한다.

- 일반 상품과 다른 연출
- 더 높은 기본 가치
- 특정 열 또는 하단 슬롯에서만 높은 확률로 등장

예시:

- 안정 열 상단: 0%
- 공격 열 상단: 2%
- 심층 열 하단: 8%
- 보너스 열 하단: 12%

### 12.2 슬롯 기반 할인권

현재 열 클리어 할인 구조는 유지하되, 일부 슬롯은 개봉 시 할인권이나 가격 관련 보정을 제공할 수 있다.

예시:

- 보너스 열 하단 상자: 다음 구매 20% 할인권
- 특정 슬롯 개봉: 무료 리롤권
- 특정 세트 타입: 다음 세트 첫 구매 할인

단, 런 보상으로서의 `특수 쿠폰 카드`와 혼동되지 않도록 별도 체계로 관리해야 한다.

## 13. 라운드 진열 시스템과의 연결

이 문서의 경제 설계는 [라운드 진열 시스템 기획.md](/f:/Workspace/BoxValueSimulator/docs/라운드%20진열%20시스템%20기획.md)와 결합될 때 완성된다.

핵심 연결은 다음과 같다.

- 상자와 열 설계는 `무엇을 먼저 열지`를 결정한다.
- 진열 슬롯 시스템은 `무엇을 남기고 무엇을 팔지`를 결정한다.

즉 전략은 두 층이다.

1. 상자 선택 전략
2. 상품 핸드 운영 전략

### 13.1 라운드 목표

- 유저는 개봉한 상품을 즉시 판매하거나 진열할 수 있다.
- 진열 슬롯은 3칸이다.
- 진열 슬롯 3개의 판매가 총합이 라운드 목표 이상이면 라운드 클리어다.

### 13.2 진열과 즉시 판매의 역할 차이

- 즉시 판매: 현금 확보
- 진열: 라운드 목표 달성

즉시 판매는 다음 상자를 열기 위한 유동성이고, 진열은 라운드 통과용 자산이다.

### 13.3 슬롯이 꽉 찼을 때

- 새 상품을 진열하려면 기존 진열 상품 중 하나를 즉시 판매해 비워야 한다.
- 즉, 패를 버리고 새 패를 잡는 핸드 교체 전략이 생긴다.

### 13.4 라운드 종료 시 진열 상품 처리

- 진열 상품은 라운드 종료 시 판매되지 않는다.
- 해당 상품은 회수된다.
- 회수의 대가로 라운드 보상으로 `특수 쿠폰 카드 선택 기회`를 획득한다.

### 13.5 초과 달성 보상

라운드 목표를 크게 초과할수록 카드 선택 리롤 기회를 얻는다.

- 100% 이상: 카드 선택 1회
- 200% 이상: 선택지 리롤 1회
- 300% 이상: 선택지 리롤 3회

즉 고가 상품 진열은 직접 돈이 아니라 `런 빌드 강화`로 환산된다.

## 14. 유저에게 보여줘야 할 정보

### 14.1 상자 선택 단계

최상단 상자에서는 아래 정보를 보여줘야 한다.

- 구매가
- 상품가 구간
- 열 타입
- 예상 회수율

## 17. 결론

현재 게임은 `10개 상자 세트와 열 해제 구조`가 핵심이다.
앞으로는 여기에 `3슬롯 진열 라운드 구조`가 더해진다.

따라서 개선의 핵심은 다음 다섯 가지다.

- 세트가 성격을 가져야 한다.
- 열이 역할을 가져야 한다.
- 아래층일수록 기대값이 올라가야 한다.
- 유저는 공개 정보만으로 어느 열을 밀지 판단할 수 있어야 한다.
- 개봉 후에는 어떤 상품을 팔고 어떤 상품을 진열할지 결정해야 한다.

이 방향으로 바꾸면 게임은 단순 상자 개봉에서 벗어나, `열 전략과 핸드 운영이 결합된 라운드형 경제 게임`이 된다.
