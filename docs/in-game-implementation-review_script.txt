인게임 구현 상태 리뷰입니다.

이 문서는 리드미에 설계된 내용을 기준으로, 현재 소스 코드의 실제 구현 상태를 비교 분석한 리뷰입니다.

먼저 한 줄 요약부터 말씀드리겠습니다.

현재 빌드는 상자 세트를 생성하고, 상자를 선택해서 구매하고, 낙하 후 개봉해서 상품을 확인하고, 판매까지 이어지는 기본 플레이 루프는 실제로 동작합니다. 그러나 리드미에서 강조한 상품 상태 시각화, 상태 기반 등급 판정, 파산 시 복구 미니게임, 그리고 시장과 보너스 같은 확장 경제 요소들은 아직 없거나 많이 단순화되어 있는 상태입니다.


다음은 현재 구현된 인게임 루프에 대한 설명입니다.

첫 번째, 세트 시작과 저장 복구입니다.

새 세트 시작, 저장 복구, 세트 교체 흐름은 메인 파일에 모두 연결되어 있습니다. 게임 상태 저장, 로드, 오프라인 보상은 게임스테이트매니저가 담당합니다. 초기 자금 십만 원, 오프라인 보상, 박스 상태 복구까지는 구현이 완료된 상태입니다.

두 번째, 박스 선택 단계입니다.

한 세트는 열 개의 박스로 생성됩니다. 세트 총가치 범위를 먼저 정하고, 각 박스 내부 가치를 분배하는 구조가 들어가 있습니다. 박스는 두 개에서 네 개의 타워로 쌓이고, 각 타워의 최상단 박스만 선택할 수 있습니다. 가격 태그와 잔액 기반 구매 가능 표시도 구현되어 있습니다.

세 번째, 구매, 낙하, 개봉, 결과 표시입니다.

박스 구매와 자금 차감, 낙하 시작은 박스 선택 이벤트에서 연결됩니다. 박스 낙하 물리, 착지 후 플레이 가능 상태 전환, 클릭 개봉은 언박싱씬에 구현되어 있습니다. 개봉 애니메이션, 컨페티 효과, 상품 상승 연출, 결과 상태 진입까지 모두 구현되어 있습니다. 결과 팝업과 판매 버튼 노출은 헤드업 디스플레이와 이벤트 시스템으로 연결됩니다.

네 번째, 판매와 다음 루프입니다.

판매 시 자금이 증가하고, 코인 연출이 재생되며, 박스가 숨겨집니다. 이후 남은 박스를 검사해서 다음 상자 또는 다음 세트로 이동합니다. 상품 판매 자체는 게임스테이트매니저의 셀 프로덕트 함수에서 상태를 처리합니다.


이어서 설계 대비 부분적으로만 구현된 항목들을 살펴보겠습니다.

첫 번째, 세트 총가치 기반 박스 분배입니다.

리드미의 핵심인 열 개 세트 총가치에서 개별 박스로 분배하는 구조는 들어가 있습니다. 다만 실제 플레이어가 추론할 수 있는 정보는 가격 태그, 박스 크기, 탑 구조 정도에 그칩니다. 리드미가 묘사한 강한 추리와 심리전 요소까지는 아직 약한 상태입니다.

두 번째, 타워 클리어 보너스입니다.

타워를 비우면 남은 최상단 박스들에 할인 보너스를 주는 시스템은 구현되어 있습니다. 리드미의 위험과 보상 구조를 살리는 방향이지만, 현재는 할인 수치만 바뀌고 플레이어에게 전달되는 피드백이 거의 없습니다.

세 번째, 데이터 기반 상품 풀과 3D 프리셋입니다.

대량 상품 데이터에 지엘비 포맷과 프리셋을 연결하는 방향은 상당 부분 반영되어 있습니다. 씨에스브이와 매니페스트 로딩, 지엘비 로딩과 캐시, 프리셋 적용 렌더링까지 구현된 상태입니다.


다음은 아직 구현되지 않았거나 축약된 부분들입니다.

첫 번째, 상품 상태 시각화입니다.

리드미는 스크래치, 먼지, 변색, 광택 저하 같은 상태값이 등급과 가격에 직접 연결된다고 설명합니다. 그러나 현재 코드는 이런 상태 파라미터를 별도로 생성하거나 시각화하지 않고, 등급만 확률 테이블에서 바로 뽑는 구조입니다.

두 번째, 상태 기반 등급 판정입니다.

리드미는 상품별 상태 파라미터와 판정 테이블 구조를 상정합니다. 현재는 카테고리를 넘기는 코드가 있지만, 실제 등급 롤 함수는 그 인자를 사용하지 않습니다.

세 번째, 파산 후 복구 미니게임입니다.

리드미에는 자금이 바닥나면 반복 노동 미니게임으로 복구하는 루프가 있습니다. 현재 구현에는 오프라인 보상은 있지만, 파산 상태 진입이나 복구 미니게임 자체는 없습니다.

네 번째, 시장, 브이아이피 상자, 연속 보너스 같은 확장 경제입니다.

리드미 하단 확장 항목에 있는 시장 변동, 브이아이피 상자, 연속 보너스, 창고와 전시 등은 아직 없습니다. 현재 시장 요소는 판매 시점의 단순 랜덤 배수 하나뿐입니다.


이어서 리뷰 관점의 핵심 차이와 리스크를 정리합니다.

첫 번째 리스크, 박스 내부 가치가 최종 판매가까지 강하게 이어지지 않습니다.

세트 총가치와 박스 내부 가치는 생성되지만, 최종 판매가는 결국 선택된 상품의 기본 가치를 중심으로 다시 계산됩니다. 즉 리드미가 말하는 세트 총가치 기반 심리전보다, 현재는 근사한 상품 뽑기에 확률형 등급 보정이 더해진 구조에 가깝습니다. 플레이어가 박스 가격과 크기에서 읽어야 할 정보량이 줄어들고, 세트 단위 숨은 총가치 개념이 경제 루프에 약하게 반영됩니다.

두 번째 리스크, 등급 시스템이 설계 대비 너무 단순합니다.

현재 등급은 모든 상품에 동일한 가중치 테이블을 씁니다. 상태 파라미터도 없어서, 같은 모델이라도 상태 차이로 가치가 크게 달라진다는 설계의 핵심 감각이 아직 부족합니다.

세 번째 리스크, 타워 클리어 보너스는 시스템은 있지만 플레이어 피드백이 약합니다.

타워를 비우면 할인 보너스가 발생하지만, 헤드업 디스플레이에서 명확히 전달되는 설명이 없습니다. 시스템은 있는데 체감이 약한 상태입니다.

네 번째 리스크, 상품 데이터 로딩 범위가 세션마다 달라집니다.

매 세션마다 씨에스브이 파일 중 하나에서 무작위로 일부만 선택해 로딩합니다. 성능 목적은 이해되지만 밸런스 재현성과 디버깅 일관성이 떨어집니다. 같은 빌드라도 새로고침할 때 상품 풀 성격이 크게 달라질 수 있어 플레이 테스트 결과 재현이 어려워집니다.


마지막으로 종합 판단과 우선순위 제안입니다.

현재 상태는 프로토타입으로서의 인게임 루프는 성립합니다. 박스 선택, 3D 개봉 연출, 판매 후 다음 루프로 이어지는 플레이 감각은 이미 확인 가능한 수준입니다.

다만 리드미가 목표로 삼는 게임의 정체성은 아직 절반 정도만 반영되어 있습니다. 구현 강점은 3D 연출, 루프 연결, 저장과 복구, 대량 지엘비 데이터 기반 구조입니다. 부족한 핵심은 상태 기반 감정평가, 세트 단위 경제 심리전, 파산 복구 루프, 그리고 장기 경제 확장입니다.

우선순위 제안입니다.

1순위로, 등급을 단순 확률이 아니라 컨디션 스테이트 묶음으로 분해하고, 판매가를 박스 내부 가치와 더 직접 연결하며, 타워 클리어 보너스를 헤드업 디스플레이에서 명확히 알리는 작업을 권장합니다.

2순위로, 자금 고갈 시 복구 미니게임 또는 최소 보정 루프를 추가하고, 시장 변동을 세션 단위 상태로 관리해 단순 랜덤보다 예측 가능한 경제로 확장하는 것을 권장합니다.

3순위로, 랜덤 씨에스브이 로딩 정책을 옵션화해서 테스트 모드와 실서비스 모드를 분리하는 것을 권장합니다.

이상으로 인게임 구현 상태 리뷰를 마칩니다.
